# PROJECT.md — מערכת ניהול ארוחות לנשים אחרי לידה

## סקירה כללית

אפליקציית ווב (PWA) לניהול שירות ארוחות עבור נשים שילדו.
האפליקציה מחליפה את כל התיאום הידני (שיחות טלפון, הודעות וואטסאפ) במערכת אוטומטית שבה כל אחת רואה מה היא צריכה לעשות.

## שפה

- הממשק כולו בעברית (RTL)
- כיוון הטקסט: right-to-left
- כל הטקסטים, כפתורים, הודעות שגיאה — בעברית

## 4 סוגי משתמשים

### 1. אדמין (מנהלת)
- מאשרת הרשמות (יולדות + מתנדבות)
- מגדירה תפריטים קבועים (ארוחת בוקר / שבת)
- קובעת לכל יולדת: כמה ימים מקבלת ארוחות (ברירת מחדל + אפשרות לשנות)
- רואה לוח בקרה מלא: מי מבשלת, מי מחלקת, מה חסר
- יכולה להתערב ולשנות שיבוצים

### 2. יולדת (מוטבת)
- נרשמת עם: שם, טלפון, כתובת, תאריך לידה
- ההרשמה ממתינה לאישור אדמין
- אחרי אישור: רואה את לוח הארוחות שלה
- רואה מי מבשלת ומי מחלקת כל יום
- מאשרת קבלת ארוחה

### 3. מבשלת (מתנדבת סוג 1)
- נרשמת עם: שם, טלפון, כתובת, שכונה
- ההרשמה ממתינה לאישור אדמין
- רואה לוח עם ארוחות פנויות לבישול
- בוחרת ארוחה + תאריך (לוקחת על עצמה)
- רואה את התפריט שצריך להכין
- מקבלת התראה יום לפני

### 4. מחלקת (מתנדבת סוג 2)
- נרשמת עם: שם, טלפון, שכונה, האם יש רכב
- ההרשמה ממתינה לאישור אדמין
- רואה לוח עם משלוחים פנויים
- בוחרת משלוח (לוקחת על עצמה)
- רואה: כתובת המבשלת + כתובת היולדת + שעת איסוף + שעת מסירה
- מאשרת ביצוע משלוח

## סוגי ארוחות

### ארוחת בוקר (יומית)
- נמסרת כל יום בבוקר
- תפריט מוגדר ע"י האדמין
- ברירת מחדל: 14 יום אחרי הלידה (האדמין יכולה לשנות ליולדת ספציפית)

### ארוחת שבת (שבועית)
- ליל שבת (יום שישי) + צהריי שבת
- תפריט מוגדר ע"י האדמין
- ברירת מחדל: 2 שבתות אחרי הלידה (האדמין יכולה לשנות)

## זרימת העבודה (Flow)

### הרשמת יולדת:
1. יולדת נרשמת באפליקציה
2. אדמין מקבלת התראה
3. אדמין מאשרת + קובעת מספר ימים/שבתות
4. המערכת יוצרת אוטומטית את כל הארוחות בלוח
5. ארוחות מופיעות כ"פנויות" למתנדבות

### תהליך ארוחה:
1. מבשלת רואה ארוחה פנויה ← לוקחת על עצמה
2. מחלקת רואה משלוח פנוי ← לוקחת על עצמה
3. יום לפני: שתיהן מקבלות תזכורת (WhatsApp/SMS)
4. מבשלת מכינה ← מאשרת "מוכן לאיסוף"
5. מחלקת אוספת ← מאשרת "נאסף"
6. מחלקת מוסרת ← מאשרת "נמסר"
7. יולדת מאשרת "התקבל"

### אם אין מתנדבת:
- ארוחה נשארת "פנויה" בלוח
- 48 שעות לפני: התראה לאדמין "ארוחה לא מכוסה"
- 24 שעות לפני: התראה דחופה לכל המתנדבות

## תפריטים

- האדמין מגדירה תפריטים קבועים
- כל תפריט: שם + רשימת פריטים + סוג (בוקר/שבת ליל/שבת צהריים)
- אפשר מספר תפריטים לכל סוג (רוטציה)
- המערכת משייכת תפריט אוטומטית או האדמין בוחרת ידנית

## התראות (WhatsApp + SMS via n8n)

- יולדת נרשמה → התראה לאדמין
- מתנדבת נרשמה → התראה לאדמין
- הרשמה אושרה → התראה למשתמשת
- מתנדבת לקחה ארוחה → התראה ליולדת
- תזכורת יום לפני → מבשלת + מחלקת
- ארוחה לא מכוסה 48 שעות לפני → אדמין
- ארוחה לא מכוסה 24 שעות לפני → כל המתנדבות
- ארוחה מוכנה לאיסוף → מחלקת
- ארוחה נמסרה → יולדת + אדמין

## מסכים

### מסך התחברות / הרשמה
- התחברות עם מספר טלפון + קוד SMS (OTP)
- הרשמה: בחירת תפקיד (יולדת/מבשלת/מחלקת) + פרטים

### דשבורד אדמין
- סטטיסטיקות: יולדות פעילות, ארוחות היום, ארוחות לא מכוסות
- רשימת הרשמות ממתינות לאישור
- לוח שבועי: כל הארוחות + סטטוס
- ניהול תפריטים
- ניהול משתמשות

### דשבורד יולדת
- לוח הארוחות שלי (השבוע הקרוב)
- כל ארוחה: סטטוס (פנוי/יש מבשלת/יש מחלקת/נמסר)
- כפתור "התקבל" לאישור

### דשבורד מבשלת
- לוח ארוחות פנויות לבישול
- "הארוחות שלי" — מה לקחתי על עצמי
- כפתור "מוכן לאיסוף"

### דשבורד מחלקת
- לוח משלוחים פנויים
- "המשלוחים שלי" — מה לקחתי על עצמי
- לכל משלוח: כתובת מבשלת + כתובת יולדת + שעות
- כפתור "נאסף" + "נמסר"

## טכנולוגיה

- **Frontend**: Next.js 14 (App Router) + Tailwind CSS
- **Backend/DB**: Supabase (PostgreSQL + Auth + Realtime)
- **Auth**: Supabase Phone Auth (OTP SMS)
- **Notifications**: n8n webhooks → WhatsApp API + SMS
- **Hosting**: Vercel
- **PWA**: next-pwa pour התקנה על הטלפון
- **שפה**: עברית בלבד, RTL

## מבנה בסיס הנתונים (Supabase)

### users
- id, phone, name, role (admin/beneficiary/cook/driver), address, neighborhood, has_car (for drivers), approved (boolean), created_at

### beneficiaries
- id, user_id, birth_date, num_breakfast_days (default 14), num_shabbat_weeks (default 2), active, start_date, end_date
- **Phase 6 :** num_adults, num_children, children_ages (int[]), is_vegetarian, spicy_level (0-2), cooking_notes
- **Phase 6 :** preferred_time_slot_id (FK → time_slots), shabbat_friday, shabbat_saturday, shabbat_kashrut

### menus
- id, name, type (breakfast/shabbat_friday/shabbat_saturday), items (text array), active, created_at

### meals
- id, beneficiary_id, menu_id, date, type, status (open/cook_assigned/cooking/ready/driver_assigned/picked_up/delivering/delivered/confirmed)
- cook_id (nullable), driver_id (nullable)
- pickup_time, delivery_time
- **Phase 6 :** time_slot_id (FK → time_slots), conflict_at

### time_slots *(Phase 6)*
- id, label, meal_type (breakfast/shabbat_friday/shabbat_saturday), pickup_time, delivery_time, max_per_slot, active, created_at

### meal_items *(Phase 6 — items Shabbat réservables individuellement)*
- id, meal_id (FK → meals), item_name, item_type (protein/side/salad/soup/dessert/other)
- cook_id (nullable FK → users), reserved_at, created_at

### feedbacks *(Phase 6)*
- id, meal_id, author_id, target_id (cuisinière ou livreuse), rating (1-5), message, sent_wa, created_at

### notifications_log
- id, user_id, message, type, read, channel (in_app/whatsapp/sms), created_at
